# 3. Designing for Security and Compliance

## 3.1 Designing for Security

- **IAM**: Implementing least privilege, custom roles, and service accounts.
- **Resource Hierarchy**: Using Organization, Folders, and Projects for policy inheritance.
- **Data Security**: Encryption at rest (CMEK, CSEK) and in transit.
- **Network Security**: Firewall rules, Cloud Armor, VPC Service Controls.

## 3.2 Designing for Compliance

- **Regulations**: GDPR, HIPAA, PCI-DSS.
- **Auditing**: Using Cloud Audit Logs and Cloud Logging.
- **Data Sovereignty**: Choosing appropriate regions for data residency.
